## 설치

### 시스템 요구사항
- Python 3.7 이상
- Windows, macOS, 또는 Linux

### 설치 단계

1. **프로젝트 다운로드**
   ```bash
   cd /path/to/orange_core
   ```

2. **패키지 설치**
   ```bash
   pip install -r requirements.txt
   ```

3. **애플리케이션 시작**
   ```bash
   python main.py
   ```

## 기본 사용법

### 인터페이스 구성

애플리케이션 창은 다음과 같이 구성됩니다:

```
┌─────────────────────────────────────────────────────┐
│  파일(F) 편집(E) 보기(V) 도움말(H)  메뉴 바         │
├─────────────┬──────────────────────────────────────┤
│             │                                      │
│  Node List  │                                      │
│   (Docking) │        Canvas (Editor Area)          │
│             │        (드래그 & 드롭 지원)          │
│             │                                      │
├─────────────┼──────────────────────────────────────┤
│               Log Panel (로그)                      │
└─────────────┴──────────────────────────────────────┘
```

### 윈도우 구성 요소

#### Node List (왼쪽)
- 사용 가능한 모든 노드를 카테고리별로 표시
- 드래그 앤 드롭으로 캔버스에 추가 가능
- 우클릭으로 컨텍스트 메뉴 표시

#### Canvas (중앙)
- 노드를 배치하고 구성하는 작업 영역
- 그리드 배경으로 정렬 지원
- 노드 이동, 선택, 삭제 가능

#### Log Panel (하단)
- 모든 작업의 로그 기록
- 에러 및 경고 메시지 표시
- 로그 초기화 버튼

## 노드 작업

### 노드 추가

#### 방법 1: 드래그 & 드롭
1. Node List에서 원하는 노드 찾기
2. 마우스로 노드를 드래그
3. Canvas에 놓기
4. 노드가 마우스 위치에 추가됨

#### 방법 2: 우클릭 메뉴
1. Node List에서 노드에 우클릭
2. "추가" 메뉴 선택
3. Canvas 중앙에 노드가 추가됨

### 노드 선택

- **단일 선택**: 노드를 클릭
- **다중 선택**: Ctrl+클릭으로 추가 선택
- **영역 선택**: 드래그하여 선택 박스 생성 (향후 지원)

### 노드 이동

1. 노드를 클릭 및 드래그
2. Canvas 상의 새로운 위치로 이동
3. 마우스 버튼 놓기
- 위치 정보는 자동으로 저장됨

### 노드 삭제

#### 방법 1: 우클릭 메뉴
1. 노드에 우클릭
2. "삭제" 메뉴 선택

#### 방법 2: Delete 키
1. 노드 선택
2. Delete 키 입력

### 노드 정보

각 노드는 다음 정보를 포함합니다:
- **제목**: 노드의 이름
- **ID**: 고유 식별자
- **카테고리**: 노드가 속한 카테고리
- **입출력 포트**: 다른 노드와의 연결점

## 파일 관리

### 새 프로젝트 만들기

**메뉴**: 파일 → 새로 만들기 (Ctrl+N)
- 확인 대화상자에서 "Yes" 선택
- 현재 Canvas 초기화 (저장하지 않은 작업 손실됨)

### 파일 저장

**메뉴**: 파일 → 저장 (Ctrl+S)
1. 저장 대화상자 열기
2. 파일 이름 입력
3. Canvas 파일 위치 선택
4. "저장" 버튼 클릭

**저장 형식**: `.canvas` 파일 (JSON 형식)

### 파일 열기

**메뉴**: 파일 → 열기 (Ctrl+O)
1. 열기 대화상자 열기
2. 이전에 저장한 `.canvas` 파일 선택
3. "열기" 버튼 클릭
4. Canvas의 모든 노드가 로드됨

### 파일 형식

Canvas 파일 구조:
```json
{
  "version": "1.0",
  "nodes": [
    {
      "node_id": 1,
      "title": "CSV File",
      "image_path": "csv_file.png",
      "category": "Data",
      "x": 100,
      "y": 150,
      "input_ports": [],
      "output_ports": ["data"]
    }
  ]
}
```

## 편집 기능

### 실행 취소 (Undo)

**메뉴**: 편집 → 실행 취소 (Ctrl+Z)
- 마지막 작업 되돌리기
- 메뉴에 되돌릴 작업명 표시

### 다시 실행 (Redo)

**메뉴**: 편집 → 다시 실행 (Ctrl+Y)
- 실행 취소한 작업 다시 실행
- 메뉴에 재실행할 작업명 표시

### Canvas 지우기

**메뉴**: 편집 → Canvas 지우기
- 확인 대화상자에서 "Yes" 선택 필요
- 모든 노드 제거 (되돌릴 수 없음)

## 단축키

| 단축키 | 기능 |
|--------|------|
| Ctrl+N | 새 프로젝트 |
| Ctrl+O | 파일 열기 |
| Ctrl+S | 파일 저장 |
| Ctrl+Z | 실행 취소 |
| Ctrl+Y | 다시 실행 |
| Delete | 선택한 노드 삭제 |
| Ctrl+Q | 애플리케이션 종료 |

## 팁과 요령

### 효율적인 작업

1. **자주 저장하기**
   - 중요한 작업 후 정기적으로 저장
   - Ctrl+S 단축키 활용

2. **Undo/Redo 활용**
   - 실수는 Ctrl+Z로 되돌리기
   - 최근 작업 여러 개 되돌릴 수 있음

3. **노드 정렬**
   - 같은 카테고리 노드를 함께 배치
   - 그리드 배경을 참고하여 정렬

4. **로그 확인**
   - 문제 발생 시 하단 Log 확인
   - 에러 메시지에서 원인 파악

### 고급 팁

1. **대량 작업**
   - 유사한 작업 여러 개는 한 번에 진행
   - Ctrl+Z로 필요하면 되돌리기

2. **프로젝트 조직**
   - 카테고리 구분이 명확하도록 배치
   - 주석이 필요한 부분은 노드 배치로 표현

3. **성능 최적화**
   - 너무 많은 노드 추가 피하기
   - 정기적으로 불필요한 노드 제거

## 문제 해결

### 노드가 추가되지 않음

**확인사항**:
- Canvas가 맨 배경에 있는지 확인
- Log 패널에 에러 메시지 확인
- 다시 한 번 시도

**해결책**:
- 애플리케이션 재시작
- 시스템 메모리 확인

### 파일을 열 수 없음

**확인사항**:
- 파일 형식이 `.canvas`인지 확인
- 파일이 손상되지 않았는지 확인
- 충분한 권한이 있는지 확인

**해결책**:
- 다른 파일로 시도
- 파일 다시 저장 시도

### 응답 없음 (Hang)

**원인**:
- 너무 많은 노드 추가
- 시스템 리소스 부족

**해결책**:
- Ctrl+C로 애플리케이션 종료
- 노드 개수 줄이기
- 시스템 재시작

## 자주 묻는 질문 (FAQ)

### Q: 노드 간 연결을 시각화할 수 있나요?
A: 현재 연결 정보는 저장되지만 시각화는 향후 지원 예정입니다.

### Q: 커스텀 노드를 추가할 수 있나요?
A: 현재는 기본 제공 노드만 사용 가능합니다. 플러그인 시스템은 향후 추가 예정입니다.

### Q: 노드의 성능 영향은 얼마나 되나요?
A: UI 성능에만 영향을 미치며, 실제 데이터 처리는 현재 미구현 상태입니다.

### Q: 최대 몇 개의 노드를 추가할 수 있나요?
A: 이론상 무제한이지만, 시스템 메모리에 따라 제한됩니다.
